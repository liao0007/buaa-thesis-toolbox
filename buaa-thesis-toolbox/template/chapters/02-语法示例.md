## 引用与脚注

- 多篇：`[@fama1983; @liweian2005]` → [@fama1983; @liweian2005]
- 中英混排：[@zhangzhiqiang2013; @knuth84]

### 脚注

本模板脚注用 **LaTeX** 写法（依赖 `raw_tex`），导出后为**按页重置的带圈序号**（①②③…），勿用 Pandoc 的 `[^id]` 语法（体例不一致）。

写法：在需标注处紧接写入：

```
正文语句。\footnote{脚注内容，可含简单说明或补充文献信息。}
```

实例：这里演示第一处脚注。\footnote{第一条脚注：用于补充定义、数据来源或不宜放入正文的说明。}同一段可再跟第二处。\footnote{第二条脚注：编号按页重新从 ① 起算，换页后再次从 ① 开始。}

注意：

- `\footnote{…}` 写在句内或句末均可，一般放在标点前或后按学院习惯统一；
- 脚注里避免复杂浮动环境（图、表）；短公式可以用 `$…$`；
- 不要在同一页堆积过长脚注，以免版心被挤乱。

## Markdown 表格

推荐管道表（不要手写 `table`/`longtable`）。Pandoc 默认输出可跨页 `longtable`；续页自动加「题注（续）」。

写法：

```
Table: 题注 {#tab:示例}

| 列一 | 列二 |
| ---- | ---- |
| …    | …    |
```

Table: 短表示例 {#tab:short-demo}

交叉引用：`\ref{tab:short-demo}` → 表\ref{tab:short-demo}。

## TikZ 图

Obsidian 用 `luatikz` 预览；PDF 由 `tikz.lua` 包装为浮动图。在代码块顶部写：

```
% caption: 题注
% label: fig:示例
```

```luatikz
% caption: 模板写作流程示意
% label: fig:workflow
\begin{tikzpicture}[
  node distance=8mm and 10mm,
  box/.style={draw, rounded corners, align=center, font=\footnotesize,
    minimum width=2.4cm, minimum height=1.0cm},
  arr/.style={-{Latex}, thick}
]
\node[box] (a) {00-meta\\元数据};
\node[box, right=of a] (b) {chapters\\Markdown};
\node[box, right=of b] (c) {Pandoc\\+ filters};
\node[box, below=of b] (d) {buaa.cls\\XeLaTeX};
\node[box, below=of d] (e) {PDF};
\draw[arr] (a) -- (b);
\draw[arr] (b) -- (c);
\draw[arr] (c) -- (d);
\draw[arr] (d) -- (e);
\end{tikzpicture}
```

引用：`\ref{fig:workflow}` → 图\ref{fig:workflow}。

## Markdown 插图

用 Pandoc 图片语法；导出时自动包成浮动图。路径相对论文根目录（如 `assets/`）。

写法：

```
![题注](assets/文件名.jpg){#fig:标签 width=90%}
```

实例：

![位图插图路径演示（mac.jpg）](assets/mac.jpg){#fig:mac-demo width=80%}

交叉引用：`\ref{fig:mac-demo}` → 图\ref{fig:mac-demo}（勿用 `@fig:…`）。

## 算法

\begin{algorithm}[htp]
  \caption{编码流程（语法演示）}
  \label{alg:demo}
  \KwData{文本片段}
  \KwResult{主题编码表}
  初始化编码簿\;
  \While{仍有未编码片段}{
    阅读当前片段\;
    \eIf{可归入已有主题}{
      标注主题编号\;
    }{
      新增主题\;
    }
  }
\end{algorithm}
